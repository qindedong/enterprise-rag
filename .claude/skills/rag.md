# RAG 流程技能 (RAG)

## 描述
当用户涉及 RAG（检索增强生成）管线的设计、实现或优化时，提供端到端的 RAG 流程指导。

## 触发条件
- 用户提到"RAG"、"检索增强生成"、"知识库问答"
- 用户需要实现或优化文档问答功能
- 用户询问 RAG 的某个环节

## RAG 完整流程

### 离线阶段（文档入库）

```
文档上传 → 格式解析 → 文本清洗 → 智能分块 → 向量化 → 向量索引存储
                                  ↓
                            元数据提取 → 关系图谱构建（可选）
```

#### 1. 文档解析
- **支持格式**：PDF、Word(.docx)、Markdown、TXT、HTML、图片(OCR)
- **解析策略**：
  - PDF: 使用 PyMuPDF 或 pdfplumber，优先提取文字层，无文字层时走 OCR
  - Word: 使用 python-docx，保留标题层级结构
  - Markdown: 按标题层级切分，保留代码块完整性
  - 图片: 使用 PaddleOCR 或 Tesseract 进行 OCR 识别

#### 2. 文本清洗
- 移除多余空白和换行
- 统一全角/半角标点符号
- 修复 PDF 提取常见的断行问题
- 过滤页眉页脚等噪声内容

#### 3. 智能分块
- **分块策略**：
  - **固定大小分块**：适合通用场景，如 chunk_size=512, overlap=64
  - **语义分块**：按段落、章节自然边界切分
  - **句子级分块**：适合 FAQ 类短文本
  - **递归分块**：先用大分隔符（标题），再逐步细化
- **关键参数**：
  - `chunk_size`：分块大小（通常 256-1024 tokens）
  - `chunk_overlap`：重叠大小（通常 chunk_size 的 10-20%）
  - `separators`：分隔符优先级列表

#### 4. 向量化
- 选择合适的 Embedding 模型（见 embedding 技能）
- 批量处理以提高效率
- 记录向量维度和模型版本

#### 5. 向量索引
- 使用 Milvus / Qdrant / Weaviate / Chroma 等向量数据库
- 建立合适的索引类型（HNSW / IVF_FLAT 等）
- 配置索引参数以平衡精度和速度

### 在线阶段（问答流程）

```
用户提问 → 查询改写 → 向量检索 → 关键词检索 → 混合融合 → 重排序 → 上下文组装 → LLM生成 → 后处理 → 返回答案
                                                              ↓
                                                          引用溯源
```

#### 1. 查询改写（Query Rewriting）
- **意图识别**：判断问题是事实查询、推理分析还是闲聊
- **查询扩展**：补充同义词、相关概念
- **多轮对话上下文融合**：将历史对话融入当前查询
- **Query2Doc**：先生成一个伪文档再检索

#### 2. 多路召回
- **向量检索（Dense Retrieval）**：语义相似度匹配
- **关键词检索（Sparse Retrieval）**：BM25 / TF-IDF 精确匹配
- **混合检索（Hybrid Search）**：融合两种结果

#### 3. 融合策略（Fusion）
- **RRF（Reciprocal Rank Fusion）**：`score = Σ 1/(k + rank_i)`
- **加权融合**：根据场景调整向量/关键词权重
- **分数归一化**：Min-Max 或 Z-score 归一化后融合

#### 4. 重排序（Reranking）
- **Cross-Encoder 模型**：如 bge-reranker、Cohere Rerank
- **LLM 重排序**：使用 LLM 评估文档相关性
- **多样性重排序**：MMR 算法避免冗余

#### 5. 上下文组装
- **Top-K 截断**：选取最相关的 K 个文档
- **相似度阈值过滤**：过滤低于阈值的文档
- **上下文压缩**：超长上下文时进行摘要压缩
- **格式化**：为每个文档添加序号、来源标记

#### 6. LLM 生成
- 使用合适的 Prompt 模板（见 prompt 技能）
- 传入组装好的上下文
- 控制输出格式（引用标记、结构化回答）

#### 7. 后处理
- **引用提取**：从回答中提取并验证引用
- **答案验证**：检查回答是否忠于上下文（Faithfulness）
- **敏感信息过滤**：移除不应展示的内容
- **格式化输出**：Markdown 渲染、代码高亮等

### 评估与反馈闭环

```
用户反馈 → 评估指标计算 → 问题诊断 → 策略调整
```

#### 关键评估指标
- **检索指标**：Recall@K、MRR、NDCG、Hit Rate
- **生成指标**：Faithfulness（忠实度）、Answer Relevance（相关性）、Context Precision/Recall
- **业务指标**：用户满意度、答案采纳率、平均响应时间
